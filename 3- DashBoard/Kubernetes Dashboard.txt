Step 1: Install Kubernetes Dashboard 

The official Kubernetes Dashboard is available via its GitHub repo. 

kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml 

This will create a namespace called kubernetes-dashboard and deploy all necessary components.



Step 2: Check the pods and services

kubectl get pods -n kubernetes-dashboard
kubectl get svc -n kubernetes-dashboard

You should see something like this: 

NAME                        TYPE        CLUSTER-IP       EXTERNAL-IP   PORT(S)   AGE
dashboard-metrics-scraper   ClusterIP   10.106.139.148   <none>        8000/TCP  2m
kubernetes-dashboard        ClusterIP   10.105.199.14    <none>        443/TCP   2m

Step 4: Create a Service Account for Dashboard Login 

Create a file named dashboard-adminuser.yaml: 


apiVersion: v1
kind: ServiceAccount
metadata:
  name: admin-user
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: admin-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard




Apply it:

kubectl apply -f dashboard-adminuser.yaml







Step 5: Get the Bearer Token 

To log in to the dashboard, you'll need the token: 




kubectl -n kubernetes-dashboard create token admin-user



This will output a long token string ‚Äî copy it and paste into the Dashboard login screen when prompted.










Optional: Use Kubeconfig File 

Alternatively, you can use your existing ~/.kube/config file to authenticate if the dashboard supports kubeconfig upload (some versions do).
